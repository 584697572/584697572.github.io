​	模板的通用并不是万能的

*c++*：面向对象编程

​	泛型编程 模板

​				函数模板

​				类模板



***函数模板***

`template<typename T>`

使用模板两种方式：

​	自动类型推导：必须推导出一致的数据类型T才可以使用（比如一个int 一个char）

​	显示指定类型

模板必须要确定T的数据类型，才可以使用

目的：提高复用性，将类型参数化

普通函数类型会发生隐式类型转换

函数模板使用显示指定类型会发生隐式类型转换

​		使用自动类型推导**不**会发生隐式类型转换

*调用规则*：

​	如果函数模板和普通函数都可以实现，优先调用普通函数

​	可以通过空模板参数列表来强制调用函数模板

​	函数模板也可以发生重载

​	如果函数模板可以产生更好的匹配，优先调用函数模板（普通函数需要进行隐式类型转换时）

​	***类模板***

`Template<class a>`

类模板和函数模板语法相似，在声明模板template后面加类，此类成为类模板

和函数模板区别：

​	类模板没有自动类型推推导

​		~~Person p(“张三”，22);~~

​		`Person <string,int>p(“张三”，22);`

​	类模板在模板参数列表中可以有默认参数

​		`Template<class a=int>`

类模板中的成员函数并不是一开始就创建的，在调用时才去创建

类模板对象做函数参数：

​	指定传入类型(最常用)

​		`void printPerson1(Person<string,int>&p)`

​	函数模板配合类模板：

​	参数模板化

​		`template<class T1,classT2>`

​		`void printPerson2(Person<T1,T2>&p)`

​	整个类模板化

​		`template<class T>`

​		`void printPerson3(T &p)`

`cout<<“T1的类型：”<<typeid(T1).name()<<endl;`

类模板继承：

​	如果父类是类模板，子类需要指定出父类中T的数据类型

​	如果想要灵活进行构建，子类也要变为类模板

```
 class base{T m;};

template<class T1,class T2>

class son::public base<T2>

{T1 m;};
```

```
//构造函数类外实现

template<class T1,class T2>

Person<T1,T2>::Person(T1 name,T2 age){}

//成员函数类外实现

template<class T1,class T2>

void Person<T1,T2>::showPerson(){}
```

类模板分为年间编写：

​	由于类模板中成员函数创建时机是在调用阶段，导致份文件编写时链接不到

​		1：直接包含.cpp源文件

​		2：将声明和实现写到同一个文件中，并更改后缀名为.hpp(约定俗成)



全局函数类内实现：直接在类内声明友元

全局函数类外实现：需要提前让编译器知道全局函数的存在
